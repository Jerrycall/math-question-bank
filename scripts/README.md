# 脚本使用说明

本目录下的脚本用于题目数据导入与同步，在项目根目录执行对应 `npm run` 命令即可。

---

## 一、import-shanghai：导入上海高三复习题集

**用途**：将「流墨轩」格式的 Markdown 题目库（如上海高三复习题集/题目库）批量导入到题库，并**保留原有标签**（知识点、方法、思想、来源、题型）。

### 使用方法

在项目根目录（`高中数学题库/`）执行：

```bash
# 方式 1：使用默认路径
npm run import-shanghai
```

默认读取的目录为：

```
/Users/zhouzhijie/cursor/流墨轩·AI教研 - 新教研员/数学-高中知识库/03-练习题库/上海高三复习题集/题目库
```

若题目在其他位置，可指定环境变量：

```bash
# 方式 2：指定题目库目录（请改为你的实际路径）
SHANGHAI_QUESTIONS_DIR="/你的/题目库/绝对路径" npm run import-shanghai
```

### 前置条件

- 已配置好数据库（`.env` / `.env.local` 中的 `DATABASE_URL`）
- 已执行过 `npm run db:push` 建表
- 题目为 `.md` 文件，且含有 YAML frontmatter（id、题型、难度、知识点、方法、思想、来源等），正文含 **题目** / **答案** / **解析** 段落

### 导入结果说明

- 每道题在题库中的 **slug** 为 `shanghai-` + 原 id（如 `shanghai-D003`、`shanghai-E001`）
- 原 frontmatter 中的 **知识点** → 知识点标签，**方法** → 方法标签，**思想** → 思想标签，**来源** → 来源标签，**题型** → 以「题型：填空题/选择题/解答题」等形式作为来源类标签保留
- 重复执行时，相同 slug 的题目会被**更新**，不会重复插入

### 跳过规则

- 若某个 `.md` 文件中没有可识别的 **题目** 段落，该文件会被跳过并在终端提示「跳过（无题目内容）」

## 二、sync-md：同步本项目 questions/ 题目

**用途**：将项目内 `questions/` 目录下的 Markdown 题目（本仓库自带格式）同步到数据库。

### 使用方法

```bash
npm run sync-md
```

会扫描 `高中数学题库/questions/` 下所有 `.md` 文件，按 frontmatter 中的 `slug`、`title`、`tags` 等写入或更新数据库。题目格式要求见项目根目录 **README.md** 中的「题目文件格式」一节。

## 三、fix-imported-images：修复已导入题目中的图片路径

**用途**：对**已经通过 import-shanghai 导入**、且题目/答案/解析里含有**相对路径图片**（如 `![](./img/1.png)`、`![](images/fig.png)`）的题目，将图片从源 Markdown 所在目录拷贝到项目的 `public/uploads/`，并把题目内容中的路径改为 `/uploads/xxx`，这样在应用里打开题目时图片可正常显示。

### 使用方法

在项目根目录执行（需已配置 `DATABASE_URL`）：

```bash
npm run fix-imported-images
```

脚本会：

1. 查询所有在库中保存了 **mdFilePath**（源 .md 路径）的题目
2. 从每道题的 content / answer / analysis 中找出相对路径的图片
3. 按 md 所在目录解析出图片的绝对路径，若文件存在则拷贝到 `public/uploads/`，命名为 `{题id}-0.png`、`{题id}-1.png` 等
4. 将题目内容中的该相对路径**全部**替换为 `/uploads/xxx`，并写回数据库

### 适用条件

- 题目在导入时写入了 **mdFilePath**（当前 import-shanghai 已写入）
- 源题目库目录与导入时一致（图片仍在原位置），否则脚本会提示「文件不存在」并跳过该图

若你修改过题目库路径，可指定与 import-shanghai 相同的环境变量（脚本主要用 mdFilePath，该变量仅作备用）：

```bash
SHANGHAI_QUESTIONS_DIR="/你的/题目库/绝对路径" npm run fix-imported-images
```

### 执行后

- 图片文件会出现在 **`public/uploads/`**
- 题目在应用中的展示会使用 `/uploads/xxx`，图片即可正常显示

## 四、seed-knowledge：整理知识点层级（一级分类 + 二级归入）

**用途**：建立一级知识点分类（集合、命题与逻辑、不等式、函数、三角、向量、复数、数列、立体几何、解析几何、导数、积分、概率统计等），并将**现有知识点标签**按名称关键词自动归入对应分类作为二级目录。

### 使用方法

在项目根目录执行（需已配置 `DATABASE_URL`）：

```bash
npm run seed-knowledge
```

脚本会：

1. **一级**：确保各分类标签存在且为根节点（无父级），并按上述顺序设置 `sortOrder`
2. **二级**：对所有其他「知识点」类型标签，根据名称关键词匹配到某一类，将其 `parentId` 设为该分类的 id；未匹配到的保持不归入（可后续在应用内手动调整）

### 一级分类顺序

| 顺序 | 分类       | 匹配关键词示例（三角、导数优先于函数） |
|------|------------|----------------------------------------|
| 1    | 集合       | 集合                                   |
| 2    | 命题与逻辑 | 命题、逻辑、充分、必要、条件、全称…   |
| 3    | 不等式     | 不等式                                 |
| 4    | 三角       | 三角、正弦、余弦、正切、诱导、恒等…   |
| 5    | 向量       | 向量                                   |
| 6    | 复数       | 复数                                   |
| 7    | 数列       | 数列、等差、等比、递推、求和           |
| 8    | 立体几何   | 立体、空间、球、柱、锥、三视图、线面… |
| 9    | 解析几何   | 解析、直线、圆、椭圆、双曲线、抛物线… |
| 10   | 导数       | 导数、导、极值、切线                   |
| 11   | 积分       | 积分、定积分、不定积分                 |
| 12   | 函数       | 函数、指数、对数、幂、单调性、奇偶…   |
| 13   | 概率统计   | 概率、统计、分布、抽样、期望、方差     |

### 执行后

- 接口 `GET /api/tags?type=KNOWLEDGE&tree=true` 会返回树形结构：11 个一级节点，其下为归入的二级知识点
- 未匹配到的知识点仍为根节点，可在题库/标签管理里手动挂到某分类下
