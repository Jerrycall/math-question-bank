# 部署与数据库同步

## Vercel 构建时自动 `prisma db push`

`package.json` 的 `build` 脚本为：`prisma db push && next build`。  
这样在每次部署时会先把 **Neon** 上的表结构推到与 `schema.prisma` 一致，避免出现「代码已更新、数据库缺列」导致的 **题集详情加载失败** 等问题。

### 必配环境变量

在 Vercel 项目 **Settings → Environment Variables** 中，为 **Production**（以及需要的 Preview）配置：

- **`DATABASE_URL`**：Neon 提供的连接串（与本地 `.env` 同类）

且该变量需对 **Build** 阶段可见（Vercel 默认同一套变量会注入到 Build，一般无需额外设置）。

若 Build 阶段没有 `DATABASE_URL`，`prisma db push` 会失败，部署会红。

## 手动同步（备选）

本地配置指向 Neon 的 `DATABASE_URL` 后执行：

```bash
npx prisma db push
```

## 安全说明

`db push` 会按 schema 改表结构；生产环境若需更严格变更流程，可改为 `prisma migrate` 并去掉 build 里的 `db push`。
