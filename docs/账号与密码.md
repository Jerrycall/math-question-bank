# 账号与密码（遗忘处理）

## 当前机制

- 账号仅有 **用户名 + 密码**，**没有绑定邮箱**，因此 **不支持「忘记密码」邮件自助找回**。
- 密码在数据库中以 **PBKDF2 哈希** 存储，任何人（含管理员）都 **看不到原密码**。

## 普通用户忘记密码

1. 联系 **管理员**（在 `ADMIN_USERNAMES` 中的账号）。
2. 管理员登录后进入 **`/admin`**，在用户列表中找到该用户，点击 **「重置密码」**。
3. 填写新密码（至少 6 位）、再次确认，并勾选确认后提交。
4. 管理员通过安全渠道（当面、私信等）将 **新密码** 告知用户。

**限制**：`ADMIN_USERNAMES` 中的 **管理员账号** 不能在后台被重置密码（避免误操作或单点被攻破后改遍全员）。若管理员忘记密码，见下文。

## 管理员忘记密码

可选方案：

1. **Neon / PostgreSQL 控制台** 执行 SQL，将 `accounts` 表中对应行的 `passwordHash` 更新为新生成的哈希（需用与项目一致的 `hashPassword` 逻辑生成，或临时注册一个小号把其 `passwordHash` 复制过去再改用户名——不推荐，易错）。
2. **本地脚本**：在项目根目录用 Node 调用 `src/lib/auth.ts` 的 `hashPassword("新密码")`，将结果写入数据库。
3. **临时处理**：在 Vercel 环境变量中 **暂时换一个管理员用户名** 为新注册账号，用新账号进后台后再处理原账号（仍须数据库改哈希才能真正恢复旧管理员用户名）。

## 未来可扩展（未实现）

- 绑定 **邮箱** 或 **手机**，实现「忘记密码 → 验证码 / 链接」自助重置。
- 使用 **Resend / SendGrid** 等发信服务，需额外配置与合规说明。
