# 题目导入与同步指南

按下面步骤操作即可完成题目导入与同步。**所有命令都在终端执行，且需先进入项目目录。**

**项目目录（请按你的实际路径修改）：**
```text
/Users/zhouzhijie/cursor/高中数学题库
```

---

## 快速：把本地数据同步到线上数据库（Neon）

### 情况 A：只增加/修改了「上海高三复习题集/题目库」里的题目（推荐，更快）

只处理该文件夹里的 Markdown，不搬整库，速度更快。

1. 获取 Neon 连接串：Vercel → math-question-bank → **Settings** → **Environment Variables** → **DATABASE_URL** → **Reveal** → 复制整段。
2. 打开终端，进入项目目录后执行（**把 `你的Neon连接串` 换成上一步复制的整段**）：
   ```bash
   cd /Users/zhouzhijie/cursor/高中数学题库
   DATABASE_URL="你的Neon连接串" npm run import-shanghai
   ```
3. 脚本只扫描并写入「题目库」目录下的 .md 文件（有则更新、无则新增），看到 `🎉 导入完成` 后刷新 Vercel 网站即可。

### 情况 B：本地数据库整体和线上要对齐（全量同步，较慢）

本地有很多来源的题目（含在应用里手动加的、或别的目录同步的），需要整库覆盖到线上时再用。

1. 获取 Neon 连接串（同上）。
2. 执行：
   ```bash
   cd /Users/zhouzhijie/cursor/高中数学题库
   TARGET_DATABASE_URL="你的Neon连接串" npm run sync-db
   ```
3. 会读取本地**全部**题目与标签并写入 Neon，数据量大时需几分钟。

## 如何获取 Neon 连接串

后面凡是需要「Neon 连接串」的地方，都按下面步骤复制。

1. 浏览器打开 **https://vercel.com**，登录。
2. 左侧点 **Projects**，进入项目 **math-question-bank**。
3. 顶部点 **Settings**。
4. 左侧点 **Environment Variables**。
5. 找到 **DATABASE_URL**，右侧点 **⋯** → **Reveal**。
6. 复制整段内容（形如 `postgresql://xxx:xxx@ep-xxx.aws.neon.tech/neondb?sslmode=require`），保存到记事本备用。  
   **注意：** 不要提交到 Git，不要发给别人。

## 场景一：把 Markdown 题目同步到「本地」数据库

**适用：** 你在 `questions/` 下新增或改了 `.md` 文件，想更新本地数据库。

1. 确认本地 PostgreSQL 已启动（或你用的本地数据库可访问）。
2. 确认项目里 `.env` 或 `.env.local` 的 `DATABASE_URL` 指向本地（例如 `postgresql://localhost:5432/...`）。
3. 打开终端，进入项目目录：
   ```bash
   cd /Users/zhouzhijie/cursor/高中数学题库
   ```
4. 执行同步：
   ```bash
   npm run sync-md
   ```
5. 看到类似 `✅ 同步成功：xxx` 和 `🎉 同步完成：x 成功，0 失败` 即表示完成。

---

## 场景二：把 Markdown 题目同步到「线上」Neon 数据库

**适用：** 希望用本地的 `questions/` 里的 Markdown，直接更新 Vercel 网站用的数据库。

1. 按本文「如何获取 Neon 连接串」拿到连接串。
2. 打开终端，进入项目目录：
   ```bash
   cd /Users/zhouzhijie/cursor/高中数学题库
   ```
3. 执行（**把下面整段 `postgresql://...` 换成你复制的连接串**）：
   ```bash
   DATABASE_URL="postgresql://neondb_owner:你的密码@ep-xxx-pooler.region.aws.neon.tech/neondb?sslmode=require" npm run sync-md
   ```
4. 等待输出结束，看到 `🎉 同步完成` 即可。刷新 Vercel 上的网站即可看到新题。

---

## 场景三：把「本地数据库里全部题目」同步到线上 Neon

**适用：** 本地数据库已经有很多题目（含标签、题目关联），想一次性全部搬到线上。

1. 确认本地数据库正在运行，且 `.env` / `.env.local` 的 `DATABASE_URL` 指向本地。
2. 按本文「如何获取 Neon 连接串」拿到连接串。
3. 打开终端，进入项目目录：
   ```bash
   cd /Users/zhouzhijie/cursor/高中数学题库
   ```
4. 执行（**把 `你的Neon连接串` 换成你复制的整段连接串**）：
   ```bash
   TARGET_DATABASE_URL="你的Neon连接串" npm run sync-db
   ```
5. 终端会依次输出：`📤 从本地数据库读取…` → `标签: xx, 题目: xx, ...` → `📥 写入目标数据库（Neon）…` → `✅ 标签` / `✅ 题目` / `✅ 题目-标签` / `✅ 题目关联` → `🎉 同步完成...`。  
   题目和关联很多时可能要几分钟，属正常。
6. 同步完成后，刷新 Vercel 上的网站即可看到全部题目。

## 场景四：更新数据库表结构（改过 schema 后）

**适用：** 你修改了 `prisma/schema.prisma`（例如新增字段、新表），需要把表结构应用到数据库。

### 更新「线上」Neon 数据库

1. 按本文「如何获取 Neon 连接串」拿到连接串。
2. 打开终端，进入项目目录：
   ```bash
   cd /Users/zhouzhijie/cursor/高中数学题库
   ```
3. 执行（**把 `你的Neon连接串` 换成你复制的整段连接串**）：

4. 同样看到 in sync 提示即表示线上表结构已更新。

## 场景六：处理题目里的图片（题目库导入后）

**适用：** 已用 `import-shanghai` 导入过「上海高三复习题集/题目库」，题目内容里含相对路径图片（如 `![](./img/1.png)`、`![[附件/xx.png]]`），需要统一拷到 `public/uploads` 并改链接为 `/uploads/xxx`。

1. 打开终端，进入项目目录：
   ```bash
   cd /Users/zhouzhijie/cursor/高中数学题库
   ```
2. 执行（会更新**当前 DATABASE_URL 指向的数据库**，一般为本地）：
   ```bash
   npm run fix-imported-images
   ```
3. 脚本会：在题目同目录、assets、附件、images、流墨轩根目录等位置查找图片 → 拷贝到 `public/uploads` → 把题目 content/answer/analysis 中的相对路径改为 `/uploads/xxx`。
4. 若要让**线上**也显示这些图：  
   - 先把本地数据库同步到 Neon（见上文「快速：情况 A 或 情况 B」），使线上题目内容含 `/uploads/xxx`。  
   - 再把图片提交部署一次：  
     ```bash
     git add -f public/uploads/*
     git commit -m "chore: 题目图片"
     git push origin main
     ```  
   等 Vercel 部署完成后，线上即可显示。

## 场景七：让线上网站能显示题目里的图片

**适用：** 题目内容里用了 `/uploads/xxx.png` 这类图片，但 Vercel 上打开不显示。

### 做法 A：把现有图片一起部署（做一次即可）

1. 打开终端，进入项目目录：
   ```bash
   cd /Users/zhouzhijie/cursor/高中数学题库
   ```
2. 把 `public/uploads/` 里的图片强制加入版本库并提交：
   ```bash
   git add -f public/uploads/*
   git status
   git commit -m "chore: 将题目图片加入部署以便线上显示"
   git push origin main
   ```
3. 等 Vercel 自动部署完成，刷新网站，题目里的图片应能显示。

### 做法 B：在 Vercel 用 Blob 存新上传的图（可选）

1. 在 Vercel 进入项目 **math-question-bank**，顶部点 **Storage**。
2. 点 **Create**，类型选 **Blob**，起名后创建，并勾选 **Production** 环境。
3. 创建完成后，项目会自动多出环境变量 **BLOB_READ_WRITE_TOKEN**（可在 **Settings → Environment Variables** 中确认）。
4. 在 **Deployments** 里对最新部署点 **Redeploy**，等部署完成。  
   之后在网站上**新上传**的图片会存到 Blob，可长期显示。

## 常用命令速查

| 你要做的事                     | 先 cd 到项目目录后执行的命令 |
|------------------------------|-----------------------------|
| 题目库（上海题集）→ 本地数据库 | `npm run import-shanghai` |
| **题目库（上海题集）→ 线上 Neon（只处理该目录，快）** | `DATABASE_URL="Neon连接串" npm run import-shanghai` |
| 题目库中相对路径图片 → 拷贝到 uploads 并更新题目链接 | `npm run fix-imported-images` |
| Markdown（questions/）→ 本地数据库 | `npm run sync-md` |
| Markdown（questions/）→ 线上 Neon | `DATABASE_URL="Neon连接串" npm run sync-md` |
| 本地数据库全量 → 线上 Neon（整库，较慢） | `TARGET_DATABASE_URL="Neon连接串" npm run sync-db` |
| 表结构应用到本地              | `npx prisma db push` |
| 表结构应用到 Neon             | `DATABASE_URL="Neon连接串" npx prisma db push` |
| 初始化知识点结构到 Neon       | `DATABASE_URL="Neon连接串" npm run seed-knowledge` |

**安全提醒：** Neon 连接串不要写进 Git、不要提交到仓库，只在本地终端或 Vercel 环境变量中使用。终端里传连接串时请用英文双引号包住整段。
