# 上传与部署备忘（本地 → 线上）

下次要「更新题库并上网」时，按下面顺序做即可。详细说明见 [题目导入与同步.md](./题目导入与同步.md)。

---

## 一、先更新本地数据库

**确认 `.env` / `.env.local` 的 `DATABASE_URL` 指向本地 PostgreSQL**（不要指 Neon）。

| 题目来源 | 命令 |
|----------|------|
| 上海高三复习题集（流墨轩 `.md`），**含图片** | `npm run 题目库:导入+图片` |
| 同上，只要文字不要图处理 | `npm run import-shanghai` |
| 自定义目录 | `SHANGHAI_QUESTIONS_DIR="/绝对路径" npm run import-shanghai` |
| 本仓库 `questions/` | `npm run sync-md` |

表未建过时：`npx prisma db push`

---

## 二、图片（线上不显示时十有八九是这一步）

- **同步脚本（含 `sync-db:incremental`）只更新数据库**，不会把图片文件传到 Vercel。
- 题目里应是 **`/uploads/文件名`** 这种路径；相对路径 / `![[...]]` 需先处理：
  - 已导入过：`npm run fix-imported-images`
  - 或直接用：`npm run 题目库:导入+图片`（导入 + 拷图到 `public/uploads` + 改链接）

**部署图片（必做）：** `public/uploads/*` 在 `.gitignore` 里，必须强制加入 Git 并推送，线上才有文件：

```bash
git add -f public/uploads/*
git status
git commit -m "chore: 题目图片"
git push origin main
```

等 Vercel 部署完再刷新网站。自检：浏览器打开 `https://你的域名/uploads/某图.png`，不应 404。

---

## 三、把本地库同步到 Neon（线上数据库）

**源** = 上面 `DATABASE_URL` 的本地库；**目标** = Neon（仅终端里用，勿提交到 Git）：

```bash
TARGET_DATABASE_URL="从 Vercel → Settings → Environment Variables 复制的 DATABASE_URL" npm run sync-db:incremental
```

- 日常：**增量**（按题目内容签名 + 标签等差异写入）。
- 要整库硬对齐：**`npm run sync-db`**（较慢）。
- 演练：`SYNC_DRY_RUN=1` 加在命令最前面。

**不要**把带密码的连接串发到聊天/群里；泄露后请在 Neon 重置密码并更新 Vercel 环境变量。

## 四、推荐一条龙（上海题集 + 图 + 库 + 部署）

## 六、易错点速记

| 现象 | 常见原因 |
|------|----------|
| 增量显示 0 变更 | 本地没先导入；或 `DATABASE_URL` 误指 Neon，源和目标同一个库 |
| 线上没图 | 没 `git add -f public/uploads` + push；或库里仍是相对路径 |
| 标签报 unique / slug | 已用脚本合并逻辑，先拉最新代码再跑 |
| 题目改了但没同步 | 先更新本地库，再跑增量；内容比对已按正文字段，不单靠 `updatedAt` |
